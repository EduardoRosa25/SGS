# SGS
Sistema de Gestão de Seguros


Estrutura das pastas ? 

# assets: 

   css (Para seus estilos, não pode ser inline).

   js (Para scripts).

   img (Para logotipos e imagens do layout).

# config:

Aqui ficará o arquivo de conexão com o Banco de Dados depois.

# pages:

Onde ficarão as telas do sistema logado (dashboard, cadastro de apólice, etc).

# includes:

Para pedaços de código que se repetem (ex: menu, rodapé).
